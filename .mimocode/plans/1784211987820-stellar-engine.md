## Step 3 — Create `core/transport.py` (LocalTransport)

**File:** `core/transport.py` (new), `core/__init__.py` (new)

Implements the `Transport` ABC from `providers/base.py` as `LocalTransport`, keeping Sprint 2 aligned with the adapter architecture. When Pipecat or Twilio transports arrive later, they implement the same interface — no refactoring needed.

```python
from providers.base import Transport

class LocalTransport(Transport):
    def __init__(self, ...): ...
    async def start(self) -> None: ...
    async def stop(self) -> None: ...
    async def read_audio(self) -> bytes: ...
    async def write_audio(self, audio_data: bytes) -> None: ...
```

### 3a. `read_audio()` — Record from mic with energy-based VAD

Purpose: Record from mic, detect silence via energy-based VAD, return WAV bytes.

Logic:
1. Open input stream at 16kHz mono int16
2. Read chunks in a loop, compute RMS energy of each chunk
3. If energy < `silence_threshold` for `silence_duration` consecutive seconds → stop
4. If `max_duration` exceeded → stop
5. Accumulate all chunks, write to WAV in-memory buffer
6. Return WAV bytes

**Format:** 16kHz, mono, 16-bit PCM WAV — matches what Groq Whisper expects (confirmed by `scripts/test_groq_stt.py` pattern).

### 3b. `write_audio()` — Play WAV bytes through the speaker

Purpose: Play WAV bytes through the speaker.

Logic:
1. Open WAV bytes with `wave.open(io.BytesIO(audio_data))`
2. Read frames, play via `sd.play()` + `sd.wait()`

**Sample rate mismatch note:** Cartesia TTS returns 44100 Hz WAV. Groq Whisper STT test uses 16000 Hz. The `write_audio()` function reads the WAV header and plays at whatever sample rate is in the file — no conversion needed for playback. The 44100 Hz WAV will play fine through sounddevice. The only place sample rate matters for STT is the recording side (we record at 16kHz).

**Note:** Sprint 2 uses non-streaming `read_audio()`/`write_audio()`. Streaming via Pipecat services will be introduced in Sprint 3 with the streaming pipeline. The `Transport` interface already accommodates both modes.

## Step 4 — Rewrite `main.py` with `run_one_turn()`

**File:** `main.py` (existing, replace contents)

**Sprint 2 scope note:** This sprint uses non-streaming `generate_speech()` (full-text TTS). Streaming via `generate_speech_stream()` will be introduced in Sprint 3 with the Pipecat streaming pipeline. The provider already exposes streaming; we intentionally defer it.

**Timing design:** `Total` is split into two meaningful sub-metrics:
- **Recording** — mic capture duration (user-controlled, not an AI metric)
- **AI pipeline** — STT → LLM → TTS (the metric that matters for latency targets)

```python
import asyncio
import time

from providers.base import Transport, STTProvider, TTSProvider
from providers.registry import get_stt_provider, get_tts_provider
from bots.healthcare import agent

FALLBACK_MSG = "Sorry, I encountered an issue. Please try again."

async def run_one_turn(
    transport: Transport,
    stt: STTProvider,
    tts: TTSProvider,
) -> bool:
    """Run one voice turn. Returns False if the user wants to quit."""
    t0 = time.perf_counter()

    # 1. Record
    try:
        audio = await transport.read_audio()
    except Exception as e:
        print(f"  [mic error] {e}")
        return True
    t_rec = time.perf_counter()

    # 2. STT
    try:
        text = await stt.transcribe(audio)
    except Exception as e:
        print(f"  [stt error] {e}")
        return True
    t_stt = time.perf_counter()

    # Print transcript for dev visibility
    print(f"  You: {text}")

    # Quit command
    if text.strip().lower() in ("quit", "exit", "stop"):
        return False

    if not text.strip():
        print("  (empty transcription, skipping)")
        return True

    # 3. LLM
    try:
        result = await agent.run(text)
        response = result.output
    except Exception as e:
        print(f"  [llm error] {e}")
        response = FALLBACK_MSG
    t_llm = time.perf_counter()

    # Print response for dev visibility
    print(f"  Bot: {response}")

    # 4. TTS (non-streaming; streaming in Sprint 3)
    try:
        audio_out = await tts.generate_speech(response)
    except Exception as e:
        print(f"  [tts error] {e}")
        return True
    t_tts = time.perf_counter()

    # 5. Play
    try:
        await transport.write_audio(audio_out)
    except Exception as e:
        print(f"  [speaker error] {e}")

    # Timing report (AI pipeline = STT + LLM + TTS)
    ai_ms = (t_tts - t_rec) * 1000
    total_ms = (time.perf_counter() - t0) * 1000
    print(f"  Timing — STT: {(t_stt-t_rec)*1000:.0f}ms | LLM: {(t_llm-t_stt)*1000:.0f}ms | TTS: {(t_tts-t_llm)*1000:.0f}ms")
    print(f"  AI pipeline: {ai_ms:.0f}ms | Total (incl. recording): {total_ms:.0f}ms")
    return True

async def main():
    from core.transport import LocalTransport

    # Initialize once — providers create SDK clients; reuse across turns
    try:
        transport = LocalTransport()
        stt = get_stt_provider()
        tts = get_tts_provider()
    except Exception as e:
        print(f"Failed to initialize providers: {e}")
        return

    await transport.start()
    try:
        print("Healthcare Bot — say 'quit' to exit")
        while await run_one_turn(transport, stt, tts):
            pass
    finally:
        await transport.stop()

if __name__ == "__main__":
    asyncio.run(main())
```

## Verification Plan

1. **Import check:** `python -c "from bots.healthcare import agent; from core.transport import LocalTransport"` — no errors
2. **Smoke test (no mic):** Run `python main.py` — should attempt to record, show `[mic error]` if no mic available, and exit gracefully
3. **Full loop:** With mic + speakers, run `python main.py`, speak a healthcare question, verify:
   - STT transcript prints (`You: ...`)
   - LLM response prints (`Bot: ...`)
   - TTS audio plays through speakers
   - Timing output shows both AI pipeline and total (incl. recording) metrics
4. **Quit command:** Say "quit" after recording — verify clean exit before LLM/TTS
5. **Empty transcription:** Stay silent — verify "empty transcription, skipping" message
6. **Guardrail check:** Ask "What medication should I take for a headache?" — verify it declines to prescribe
7. **Latency baseline:** Note whether AI pipeline time is within the 1–2s target (PRD §7)
8. **Provider config failure:** Start with an invalid/missing API key and verify initialization fails with a clear error message, not an unhandled traceback

## Risks / Open Questions

| Risk | Mitigation |
|---|---|
| `sounddevice` may not install on Windows without PortAudio DLL | Ship with error message pointing to PortAudio install; alternatively fall back to `pyaudio` |
| Energy-based VAD may cut off speech too early | Tune `silence_threshold` and `silence_duration` after initial testing; Sprint spec acknowledges this may need Silero/WebRTC VAD later |
| Sample rate mismatch (44100 TTS vs 16000 STT) | Not an issue for this sprint — recording is 16kHz, playback plays whatever rate is in the WAV header |
| `agent.run()` vs `agent.run_sync()` | PydanticAI `Agent.run()` is async; use `await agent.run(text)` inside the async `run_one_turn()` |
