## Step 3 — Create `core/transport.py`

**File:** `core/transport.py` (new), `core/__init__.py` (new)

### 3a. `record_until_silence()`

Purpose: Record from mic, detect silence via energy-based VAD, return WAV bytes.

```python
import io
import time
import wave
import numpy as np
import sounddevice as sd

def record_until_silence(
    sample_rate: int = 16000,
    channels: int = 1,
    dtype: str = "int16",
    silence_threshold: float = 500,   # RMS energy threshold
    silence_duration: float = 1.5,    # seconds of silence before stopping
    max_duration: float = 30.0,       # hard cap
) -> bytes:
```

Logic:
1. Open input stream at 16kHz mono int16
2. Read chunks in a loop, compute RMS energy of each chunk
3. If energy < `silence_threshold` for `silence_duration` consecutive seconds → stop
4. If `max_duration` exceeded → stop
5. Accumulate all chunks, write to WAV in-memory buffer
6. Return WAV bytes

**Format:** 16kHz, mono, 16-bit PCM WAV — matches what Groq Whisper expects (confirmed by `scripts/test_groq_stt.py` pattern).

### 3b. `play()`

Purpose: Play WAV bytes through the speaker.

```python
import io
import wave
import sounddevice as sd

def play(audio_data: bytes) -> None:
```

Logic:
1. Open WAV bytes with `wave.open(io.BytesIO(audio_data))`
2. Read frames, play via `sd.play()` + `sd.wait()`

**Sample rate mismatch note:** Cartesia TTS returns 44100 Hz WAV. Groq Whisper STT test uses 16000 Hz. The `play()` function reads the WAV header and plays at whatever sample rate is in the file — no conversion needed for playback. The 44100 Hz WAV will play fine through sounddevice. The only place sample rate matters for STT is the recording side (we record at 16kHz).

## Step 4 — Rewrite `main.py` with `run_one_turn()`

**File:** `main.py` (existing, replace contents)

```python
import asyncio
import time

from core.transport import record_until_silence, play
from providers.registry import get_stt_provider, get_tts_provider
from bots.healthcare import agent

async def run_one_turn() -> None:
    stt = get_stt_provider()
    tts = get_tts_provider()
    t0 = time.perf_counter()

    # 1. Record (blocking → run in thread)
    audio = await asyncio.to_thread(record_until_silence)
    t1 = time.perf_counter()

    # 2. STT (async)
    text = await stt.transcribe(audio)
    t2 = time.perf_counter()

    # 3. LLM (async via PydanticAI)
    result = await agent.run(text)
    response = result.output  # AgentRunResult.output → str
    t3 = time.perf_counter()

    # 4. TTS (async)
    audio_out = await tts.generate_speech(response)
    t4 = time.perf_counter()

    # 5. Play (blocking → run in thread)
    await asyncio.to_thread(play, audio_out)
    t5 = time.perf_counter()

    print(f"  STT:    {(t2-t1)*1000:.0f}ms")
    print(f"  LLM:    {(t3-t2)*1000:.0f}ms")
    print(f"  TTS:    {(t4-t3)*1000:.0f}ms")
    print(f"  Total:  {(t5-t0)*1000:.0f}ms")

async def main():
    print("Healthcare Bot — say 'quit' to exit")
    while True:
        await run_one_turn()

if __name__ == "__main__":
    asyncio.run(main())
```

## Verification Plan

1. **Import check:** `python -c "from bots.healthcare import agent; from core.transport import record_until_silence, play"` — no errors
2. **Smoke test (no mic):** Run `python main.py` — should attempt to record, show timing output even if mic is unavailable (sounddevice will raise an error we can catch)
3. **Full loop:** With mic + speakers, run `python main.py`, speak a healthcare question, verify:
   - STT text appears in console
   - LLM response is generated
   - TTS audio plays through speakers
   - Timing printout shows all four values (STT, LLM, TTS, Total)
4. **Guardrail check:** Ask the bot "What medication should I take for a headache?" — verify it declines to prescribe and recommends consulting a doctor
5. **Latency baseline:** Note whether total turn time is within the 1–2s target (PRD §7). This is the "rough timing check" from the sprint spec.

## Risks / Open Questions

| Risk | Mitigation |
|---|---|
| `sounddevice` may not install on Windows without PortAudio DLL | Ship with error message pointing to PortAudio install; alternatively fall back to `pyaudio` |
| Energy-based VAD may cut off speech too early | Tune `silence_threshold` and `silence_duration` after initial testing; Sprint spec acknowledges this may need Silero/WebRTC VAD later |
| Sample rate mismatch (44100 TTS vs 16000 STT) | Not an issue for this sprint — recording is 16kHz, playback plays whatever rate is in the WAV header |
| `agent.run()` vs `agent.run_sync()` | PydanticAI `Agent.run()` is async; use `await agent.run(text)` inside the async `run_one_turn()` |
